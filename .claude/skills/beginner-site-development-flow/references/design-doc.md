# DESIGN.md reference

ユーザーから参考URL、参考画像、またはデザインの方向性を確認できたら、マークアップ調整前にプロジェクト直下へ `DESIGN.md` を作成する。

## 目的

- デザインの方向性を後から見返せるようにする。
- 別のAIエージェントが作業しても、同じ方針で調整できるようにする。
- 参考サイトや画像の「雰囲気」は参考にするが、丸ごとコピーしないことを明確にする。

## 作成タイミング

次のいずれかが確認できたら作成する。

- 参考URLがある
- 参考画像がある
- ユーザーがデザインの方向性を言語化した
  - 例: `シンプル`, `やさしい`, `高級感`, `信頼感`, `ポップ`, `和風`, `ミニマル`

## 書く内容

```md
# Design Direction

## サイト概要

- サイト名: ...
- サイト内容: ...
- 説明文: ...

## 参考デザイン

- URL: ...
- 画像: ...
- メモ: ...

## デザインの方向性

- 雰囲気: ...
- 色: ...
- 余白: ...
- 文字: ...
- 写真・画像の使い方: ...

## 作成するページ・セクション

- ...

## 実装メモ

- 既存のAstro構成を崩さない。
- 参考デザインは雰囲気だけ参考にし、コピーしない。
- 初心者向けに、複雑な機能は追加しない。
```

## 注意

- URLや画像をそのまま素材として無断利用しない。
- 参考デザインのHTML/CSSをコピーしない。
- 個人情報、住所、電話番号、非公開情報はユーザーが明示的に掲載希望した場合だけ扱う。
